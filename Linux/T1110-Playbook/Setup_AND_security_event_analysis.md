## Setup and Reconnaissance

### Environment Overview

*   **Attacker:** Pop!_OS (Linux)
*   **Target:** Windows 10
*   **Hypervisor:** virt-manager (QEMU/KVM)
*   **Network Mode:** NAT (Windows VM visible only to Pop!_OS)

### Network Overview

```text
External Network / Internet
    └── Pop!_OS (Attacker)
          └── Virtual NAT Network (192.168.122.0/24)
                └── Windows 10 (Victim)
```

*   Pop!_OS has two network interfaces: a physical one (Wi-Fi/Ethernet) and a virtual bridge (`virbr0`) created by KVM/QEMU.

## Reconnaissance

**Commands used:**

```shell
# I'm using NAT connection in my VM WIN10 (virbr0)
ip a 
```

## Scanning & Enumeration

```shell
## Host detection, OS detection, Port Scanning 
nmap 192.168.122.0/24
nmap 192.168.122.140 -O -osscan-guess
# to scan all ports as port 5985 does not comes under top 1000 ports
nmap 192.168.122.140 -sV -p- 
```

![os detection](./img/recon&scanning2.png)
![port scanning](./img/recon&scanning3.png)

### Conclusion

*   **Port 5985 (HTTP)** is open, indicating the Windows Remote Management (WinRM) service is active. This is a common target for lateral movement.
* WinRM allows remote PowerShell execution. If authentication is weak, attackers can get full interactive shell access.

#### Logon Type

**The Authentication Core: 4624 & 4625**

These two Event IDs are the most critical in a Windows environment, logging every login attempt.

1.  **Event ID 4625 (Failed Logon)**
    This event is generated every time an account fails to log on to the local computer.

    | Field                 | Why it Matters                                                              | SOC Use Case                                                                                             |
    | :-------------------- | :-------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------- |
    | Account Name          | The user account the attacker tried to guess.                               | Brute Force/Password Spraying: Track the unique users attempted.                                         |
    | Source Network Address | The IP address the logon attempt came from.                                 | Containment: This is the IP you recommend blocking (as in your SOAR project).                            |
    | Logon Type            | Tells you how the login was attempted (see below).                          | Attack Classification: Was it local, remote, or a service?                                               |
    | Status / Sub Status Codes | The exact reason for the failure. This is the gold standard of specificity. | Tuning Alerts: Use these codes to filter out false positives.                                            |

    **Important Sub Status Codes for SOC:**

    | Code       | Description                                            | Threat Context                                                                                             |
    | :--------- | :----------------------------------------------------- | :--------------------------------------------------------------------------------------------------------- |
    | 0xC000006A | User logon with misspelled or bad password.            | High Confidence Brute Force. The user name is known/valid, but the password is wrong.                      |
    | 0xC0000064 | User name does not exist (misspelled or bad user ID).  | Password Spraying / Dictionary Attack targeting random accounts.                                           |
    | 0xC0000234 | User logon with Account Locked.                        | Confirms a brute force attack was successful in triggering the lockout policy.                             |
    | 0xC0000072 | User logon with Administratively Disabled account.     | Insider Threat/Compromise: Someone (internal or external) is actively testing disabled accounts.          |

2.  **Event ID 4624 (Successful Logon)**
    This event is generated every time an account successfully logs on to the local computer.

    | Field                 | Why it Matters                                                                                                                            | SOC Use Case                                                                                                                            |
    | :-------------------- | :---------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------- |
    | Logon Type            | Critical for determining the nature of access (see table below).                                                                          | Anomaly Detection: Flag a user logging in interactively (Type 2) for the first time on a Domain Controller.                             |
    | New Logon ID          | A unique value that ties this successful logon to other events (like a subsequent logoff or process creation).                            | Correlation: Use this ID to track everything the user did during that session.                                                          |
    | Source Network Address | Where the successful login originated.                                                                                                    | Geographic Anomaly: A successful login from India, then 5 minutes later from Brazil.                                                    |
    | Elevated Token        | Indicates if the user logged in with Administrator privileges (if UAC is enabled, an elevated token is granted).                          | Privilege Escalation: Track successful logins that gain high privileges.                                                                |

3. ** Logon Types: The 'How' of Authentication**

   The Logon Type field (found in both 4624 and 4625) is essential because it tells you the vector of the logon.
   
   | Type Code | Description         | SOC Significance                                                                                                                            |
   | :-------- | :------------------ | :------------------------------------------------------------------------------------------------------------------------------------------ |
   | 2         | Interactive         | Physical login at the keyboard or via the console. Suspicious on a server.                                                                  |
   | 3         | Network             | The most common type. Used for accessing network shares, printers, WinRM/WinRS, and most IIS/web logons.                                    |
   | 4         | Batch               | Used by scheduled tasks. Suspicious if used by a normal user account.                                                                       |
   | 5         | Service             | Used by service accounts to start a service. Red flag if a service account is used for Type 2 or 10.                                        |
   | 10        | RemoteInteractive   | Remote Desktop (RDP) connection. Crucial for detecting lateral movement and attacker control.                                               |
   | 7         | Unlock              | Workstation unlock. Not usually a security event unless paired with other suspicious activity.                                              |
